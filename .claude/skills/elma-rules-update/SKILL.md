## Процесс

### 0. Первый запуск (если нет `.env`)
- Создать `.env` в корне проекта:
  ```
  ELMA_TOKEN=2ad06f66-6ecc-42b3-9bf4-1ef21eabb371
  ELMA_BASE_URL=https://dlqixw6ehyxiy.elma365.ru/pub/v1/app
  ```

### 1. Запрос к API
- Прочитать `ELMA_TOKEN` из `.env`
- Выполнить `Invoke-RestMethod` через PowerShell с заголовком `Authorization: Bearer <токен>`
- **Если ошибка** — показать статус-код и тело ответа. **Стоп.**
- **Если успех** — распарсить `result.result`:
  - показать количество записей
  - показать полный список полей (ключи первого объекта)
  - показать 2-3 примера (ID + 2-3 ключевых поля)

### 2. Параллельный анализ (запускать одновременно)

**2а. Субагент `data-analyst` — генерация названия и описания**

Запустить агент `data-analyst` в фоне (`run_in_background: true`) со следующим промптом:

```
Задача: предложи русское название и описание для новой сущности ELMA365.

Entity path: {entity_path}
Количество записей: {count}

Список полей:
{field_list}

Примеры записей (2-3 штуки):
{sample_records_json}

Верни строго в формате:
НАЗВАНИЕ: <2-5 слов на русском>
ОПИСАНИЕ: <1-2 предложения: что хранится, для чего используется, какие ключевые поля>

Опирайся на названия полей и значения из примеров. Используй терминологию строительного бизнеса.
```

**2б. Основной агент — поиск связей по ID**

Одновременно с запуском субагента выполнить поиск связей:
- Для каждого поля собрать массив уникальных непустых значений
- Для каждой известной сущности из `CLAUDE.md` загрузить локальный файл из `data/`
- Подсчитать процент совпадений с `__id`
- Вывести поля с совпадениями **≥80%**:
  ```
  поле "tender" → tender.__id: 45/50 (90%)
  поле "developer" → _companies.__id: 26/26 (100%)
  ```
- Зафиксировать поля **без найденных связей** (для ручного уточнения)

### 3. Подтверждение Виктора

Дождаться результата субагента из шага 2а. Показать сводку и ждать ответа:

- **URL:** `<ссылка>`
- **Название (от data-analyst):** `<предложенное название>`
- **Описание (от data-analyst):** `<предложенное описание>`
- **Записей:** `<число>`
- **Найденные связи:** список из шага 2б
- **Поля без связей:** спросить у Виктора

Если Виктор подтвердил → шаг 4. Если поправил — использовать его вариант.

### 4. Запись в CLAUDE.md
- Прочитать `CLAUDE.md`, найти блок **«Уже сохранённые запросы»**
- Добавить новую запись в формате:
  ```
  - `{entity_key}` — `{URL}` — {название}: {описание}. Содержит ссылки на {связанные сущности}
  ```
- Найти блок **«Связи между сущностями»**
- Добавить новые связи в формате:
  ```
  - `{entity}.{field}` → `{target}.__id` — {описание связи}
  ```

### 5. Загрузка данных
- Запустить `.\scripts\run.ps1 load {entity_id}`
- Сообщить Виктору результат
