# График витражей — Web-интерфейс

Веб-сервер для формирования Excel-графиков производства витражей на основе данных из ELMA365.

## Быстрый старт (Windows)

```powershell
.\run_server.ps1
```

Сервер запустится на `http://localhost:8000`. Логин по умолчанию: `admin` / `admin`.

## Быстрый старт (Linux/Docker)

```bash
# Скопировать .env из корня проекта
cp ../.env .env

# Запуск через Docker
docker build -t grafikk .
docker run -p 8000:8000 -v /путь/к/ELMA_Connector/data:/app/../data grafikk
```

## Безопасность

- **Токен ELMA** — хранится только в `.env` на сервере, никогда не отдаётся фронтенду.
- **Аутентификация** — JWT-токены, пароли в SQLite (PBKDF2).
- **Пользователи** — индивидуальные логины, админ создаёт учётки через интерфейс.
- **Данные** — сервер читает локальные JSON-файлы из `../data/`, в ELMA ходит только по кнопке «Обновить» или по расписанию.

## API (для интеграций)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/login` | Получить JWT |
| GET | `/api/objects?q=текст` | Поиск объектов |
| GET | `/api/objects/{id}/korpuses` | Корпусы объекта |
| POST | `/api/generate-excel` | Сформировать Excel |
| POST | `/api/update` | Обновить данные (admin) |
| GET | `/api/users` | Список пользователей (admin) |
| POST | `/api/users` | Создать пользователя (admin) |

## Требования

- Python 3.10+
- Данные из ELMA (JSON-файлы в `../data/`)
