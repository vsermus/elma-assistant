# Подправила подпроекта: Конструктор дашбордов

## Цель
Создание веб-интерфейса для гибкого выбора метрик и построения интерактивных дашбордов на основе выбранных данных из ELMA365.

## Общие положения
- Все файлы подпроекта хранятся в папке `dashboard_constructor`.
- Основные правила проекта (из `CLAUDE.md`) остаются в силе, если они не противоречат данным правилам.

## Технический стек
- **Backend**: Легкий сервер на Python (FastAPI/Flask) для:
    - Раздачи JSON-данных из папки `data/`.
    - Запуска скриптов обновления данных (`load_data.py` или `run.ps1`).
- **Frontend**: Одностраничное приложение (HTML/CSS/JS), работающее в браузере.

## Работа с данными
- Используются только согласованные сущности: `tender`, `raboty_po_tenderu`, `spravochnik_id`, `_companies`, `users`, `statusy_rabot_po_tenderam`.
- Данные для интерфейса берутся из локальных файлов в `data/` (через сервер), а не напрямую из API ELMA, чтобы обеспечить скорость работы.
- Кнопка «Обновить» в интерфейсе должна инициировать процесс загрузки данных через сервер.

## Интерфейс и UX
- Для Виктора (аналитика): использовать терминологию Excel и Power BI (метрики, срезы, показатели).
- Короткие, понятные пояснения технических решений.
- Никаких круговых диаграмм (согласно общим правилам).
