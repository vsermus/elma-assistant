# Как работаем

1. **Новый запрос к ELMA**: я даю ссылку на API. Ты проверяешь её (выполняешь запрос). Если ELMA ответила успешно — сообщаешь мне и спрашиваешь подтверждение: название запроса, ссылку, что ещё сочтёшь нужным. После моего одобрения добавляешь в правила.

2. **Связи между запросами**: ищи совпадения автоматически по ID-полям. Например, `_id` в одном запросе может быть полем `zadanie_na_kmd` в другом. Если не можешь найти связь — спроси меня: «Виктор, не могу сопоставить поле X из запроса Y с полем из запроса Z, подскажи».

3. **Перевод заголовков**: все поля в ELMA на английском или транскрипции — в дашбордах и выгрузках показывай на русском.

4. **Проверка результата**: дашборд открывается локально в браузере. Отчитываться списком изменённых файлов не нужно, если я не попросил.

5. **Описание запросов**: каждый запрос к ELMA должен иметь русское название и описание — что именно спрашивается, какие данные возвращаются. Это нужно для AI-агента, чтобы он мог сопоставлять вопросы пользователей с нужными данными.

6. **Проверка существующих запросов**: при добавлении/обновлении правил про запросы — я по очереди предлагаю дополнить каждый существующий запрос русским названием и описанием, ты подтверждаешь или корректируешь.

# Частые задачи

- **Новый запрос к ELMA**: проверить ссылку → если успех → спросить подтверждение у меня → добавить в `config/entities.json` (id, url, name, description; если запрос роутируемый — добавить `routing_hints`: короткие ключевые слова через запятую) и сюда в список запросов ниже. Роутируемые — те, у которых есть агрегатор в `aggregator.py` и id в `ROUTABLE_CATEGORIES` (`claude_client.py`). Бот подхватит новую категорию автоматически — код трогать не нужно.
- **Загрузка данных**: `.\scripts\run.ps1 load` (все) или `.\scripts\run.ps1 load <id>` (конкретный).
- **Проверка изменений**: после загрузки запускать `.\scripts\run.ps1 watch` или `python scripts/check/check_changes.py` — сверяет слепки, выявляет новые поля и изменения записей. Автоматически запускается агентом `data-watcher`.
- **Сборка дашборда**: взять данные из одного или нескольких связанных запросов → связать по ID → собрать HTML/CSS/JS → перевести заголовки на русский → сохранить файл в проект.
- **Выгрузка в Excel**: взять данные из ELMA → сформировать Excel-файл (.xlsx).

# Инструменты и среда

**API ELMA365:**
- Базовый URL: `https://dlqixw6ehyxiy.elma365.ru/pub/v1/app`
- Токен (Bearer): `ELMA_TOKEN` из `.env` в корне проекта — единый для всех запросов
- Параметр: `query` с JSON вида `{"size": 10000}` (при необходимости URL-кодировать)
- Метод: `Invoke-RestMethod` в PowerShell с заголовком `Authorization: Bearer <токен>`
- **Важно**: в данных есть soft-deleted записи с заполненным полем `__deletedAt`. При обработке всегда исключать такие записи — учитывать только те, у которых `__deletedAt == null/None`.

**Три типа URL:**

1. **Прикладные сущности** — `.../tender/{entity}/list`
   Пример: `.../tender/zadanie_na_kmd/list?query={"size":10000}`

2. **Системные справочники** — `.../_system_catalogs/{entity}/list`
   Пример: `.../_system_catalogs/spravochnik_id/list?query={"size":10000}`

3. **Пользователи** — `.../user/list`
   Пример: `.../user/list?query={"size":10000}`

**Конфиг запросов:** `config/entities.json` — id, url, описание. Служит единым источником для загрузчика и AI-агента.

**Загрузка данных:** `python scripts/load/load_data.py` (или `.\scripts\run.ps1 load <id>`).

# Прочее

- `query`-параметр в URL: при ручном запросе через PowerShell используй `[System.Uri]::EscapeDataString()`. В `load_data.py` кодирование автоматическое.
- Имена файлов дашбордов — по-русски, транслитом или по-английски — как скажу.
- **Power Query — индексация списков**: для доступа к элементу списка по индексу использовать `_{0}`, а не `_[0]`. Поле записи после этого: `_{0}[fieldname]`. Пример: `_{0}[name]`, `_{0}[tel]`. `_[0]` — синтаксическая ошибка в M.
- **Power Query — Web.Contents для Report Server**: URL передавать статической строкой прямо в аргумент `Web.Contents(...)`, без переменных и конкатенации. Иначе Power BI Report Server не сможет обновлять данные. Токен вставлять прямо в строку `Authorization`.
